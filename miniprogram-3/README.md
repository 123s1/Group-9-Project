# 安心接送 - 家长端

面向家长用户的儿童上下学接送服务微信小程序。

## 项目简介

「安心接送」家长端微信小程序，为家长提供便捷的儿童接送预约服务，包括创建订单、实时追踪、安心日记、孩子档案管理等功能。

## 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | TypeScript (strict 模式) |
| 样式 | Less |
| 渲染 | Skyline 渲染引擎 |
| 基础库 | 微信小程序 SDK 2.32.3 |
| 后端 | 自定义 RESTful API（非云开发） |

## 快速开始

### 环境要求

- 微信开发者工具
- 项目 AppID：`wxa44f177469039748`

### 启动步骤

1. 打开微信开发者工具
2. 导入项目根目录 `miniprogram-3`
3. 填入 AppID
4. 点击编译预览

### 后端配置

修改 `miniprogram/utils/request.ts` 中的 `BASE_URL` 为实际后端地址：

```typescript
const BASE_URL = 'https://your-api-server.com'; // 替换为实际地址
```

## 核心功能

### 用户登录

- 手机号 + 短信验证码登录
- 手机号 + 密码登录
- 微信一键登录（wx.login）

### 首页

- 当前进行中订单卡片（含接送员信息）
- 快捷服务入口：立即预约、紧急加单、订单历史、孩子管理
- 会员等级卡片（银卡/金卡/钻石卡）
- 热门推荐增值服务：作业辅导、营养加餐、临时托管、周末研学
- 最近安心日记入口
- 紧急求助（平台热线 + 110）

### 订单管理

- **创建订单**：选择孩子、接送类型（上学/放学）、时间、地点、增值服务、套餐（单次/周套餐/月套餐），实时计算价格
- **订单列表**：进行中/已完成/已取消三 Tab 切换，支持下拉刷新
- **支付**：微信支付（接口已预留）

### 订单状态

```
待派单 → 已派单 → 接送员已出发 → 已到达学校 → 已接到孩子 → 途中 → 已送达 → 已完成
                                ↕
                             已取消
```

### 安心日记

- 视频日记列表（封面、标题、类型标签、接送员信息）
- 视频播放器（含弹幕功能）
- 分享给好友

### 孩子管理

- 孩子档案 CRUD（姓名、性别、学校、年级、生日、兴趣标签）
- 个性化兴趣推荐系统：基于标签匹配兴趣班、绘本、周末活动

### 个人中心

- 用户信息展示
- 会员等级与消费进度
- 订单统计
- 钱包与优惠券
- 设置、退出登录

### 投诉售后

- 投诉类型：服务态度、质量、延误、安全隐患、费用问题等
- 关联订单 + 图片上传（最多3张）
- 历史投诉记录查询

## 项目结构

```
miniprogram/
├── app.ts                     # 应用入口
├── app.json                   # 全局配置
├── app.less                   # 全局样式
├── components/                # 自定义组件
│   └── navigation-bar/        # 自定义导航栏
├── pages/
│   ├── login/                 # 登录
│   ├── index/                 # 首页 (TabBar)
│   ├── order/
│   │   ├── order/             # 订单列表 (TabBar)
│   │   └── create-order/      # 创建订单
│   ├── profile/               # 个人中心 (TabBar)
│   ├── payment/               # 支付
│   ├── safe-diary/            # 安心日记
│   ├── complaint/             # 投诉售后
│   └── children/              # 孩子管理
└── utils/
    ├── request.ts             # HTTP 请求封装
    └── api.ts                 # 7大 API 模块定义
        ├── userApi            # 用户相关
        ├── orderApi           # 订单相关
        ├── trackApi           # 实时追踪
        ├── paymentApi         # 支付与会员
        ├── valueAddedApi      # 增值服务
        ├── afterSaleApi       # 售后投诉
        └── emergencyApi       # 紧急求助
```
